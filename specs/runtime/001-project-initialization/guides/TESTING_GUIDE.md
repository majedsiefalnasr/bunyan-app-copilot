## Prerequisites

- Docker & Docker Compose (recommended) or local services (MySQL, Redis)
- PHP 8.2+, Composer installed
- Node 18+, npm (or pnpm)

Quick setup (Docker recommended):

```bash
# from repository root
docker-compose up -d
# backend
cd backend && composer install && cp .env.example .env && php artisan key:generate && php artisan migrate --seed
# frontend
cd frontend && npm install && cp .env.example .env
```

## Running Tests

### Backend — Unit & Feature Tests

Run all backend tests:

```bash
cd backend
php artisan test
# or (with more detail)
./vendor/bin/phpunit --testdox
```

Run only Feature tests:

```bash
php artisan test --testsuite=Feature
```

Run static analysis and lint:

```bash
cd backend
composer run lint   # Pint / php-cs-fixer
vendor/bin/phpstan analyze
```

### Frontend — Unit & E2E

Run unit tests:

```bash
cd frontend
npm run test
```

Run Playwright E2E smoke test:

```bash
cd frontend
npm run test:e2e
```

Run lint and typecheck:

```bash
npm run lint
npm run typecheck
```

### Full test run (both stacks)

```bash
# backend
(cd backend && php artisan test)
# frontend
(cd frontend && npm run test)
```

## Troubleshooting

- If PHP tests fail with DB errors, ensure `.env` is configured or run tests with in-memory sqlite as configured in `phpunit.xml`.
- If frontend tests fail due to missing env vars, copy `frontend/.env.example` to `.env` with `API_BASE_URL` pointing to backend.
